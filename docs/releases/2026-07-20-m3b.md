## Summary

M3B adds a deterministic Browser capability without adding arbitrary web access. Agents can open a canonical page and follow an opaque link on a bundled synthetic HTTPS site. Browser calls participate in the same action budget, trajectory, revocation, drain, adapter-close, terminal-close, and hidden-verifier lifecycle as the existing workspace, terminal, and Research capabilities.

## Added

- A Playwright `1.61.0` Browser MCP server with exactly two tools:
  - `open_page`
  - `click_link`
- A versioned offline site fixture whose ordered HTML is bound to a canonical SHA-256 manifest.
- Validation of the exact MCP server name, tools capability, tool inventory, schemas, annotations, metadata, and bounded results.
- Canonical path and opaque link-ID validation with hostile exact-type tests.
- A Browser tool facade that shares the episode action budget and emits payload-free trajectory details.
- M3B task `browser-evaluation-001`, which proves a causal Browser → Docker terminal → workspace artifact → hidden-verifier path.
- Success-path process-tree tests continuously observe the real MCP worker, Playwright driver, and Chromium descendants; the timeout-path test continuously observes a fixed hanging child with real Playwright and Chromium and requires the complete observed tree to be reaped.
- A production image containing the Playwright-managed Chromium revision in the non-root runtime user's standard cache.
- A network-disabled, read-only, capability-dropped Compose Browser smoke.
- Hosted CI coverage for the real Browser MCP, Docker integration, production image, and both Compose smokes.

## Security boundary

M3B intentionally does **not** expose:

- arbitrary URLs or general network browsing;
- selectors, forms, downloads, screenshots, cookies, storage, or page evaluation;
- caller-controlled commands, executables, browser paths, launch arguments, or site directories;
- page payloads, hidden oracle values, local paths, or raw Playwright diagnostics in trajectories.

The browser context fulfills only exact bundled pages from the synthetic origin, blocks service workers, and aborts every other request. Each admitted transport call owns one MCP stdio process and one Chromium lifecycle and closes before returning. Timeout cleanup terminates the worker process tree.

## Lifecycle and reward semantics

Browser calls consume the existing shared action budget. Episode shutdown revokes admission and drains in-flight Browser, Research, workspace, and terminal calls before adapter close and deterministic hidden verification. M3B does not add Browser-specific reward algebra or test-specific reward overrides.

## Review fixes included

Independent pre-push review identified and resolved four issues before merge:

1. cleanup now propagates non-`Exception` control-flow signals after attempting every cleanup step while preserving the original primary failure for ordinary cleanup errors;
2. documentation now distinguishes invalid calls rejected before transport admission from admitted per-call MCP/Chromium lifecycles;
3. oracle-leakage tests inspect structured model values rather than JSON text that could hide newlines through escaping;
4. timeout reaping tests continuously observe descendants instead of taking a one-time PID snapshot.

## Known limitations and deferred work

- Chromium duration is bounded by deadlines and process-tree cleanup, but M3B does not claim hard host memory or PID isolation.
- CAMEL integration, model inference, RL training, and arbitrary web browsing remain unimplemented.
- The Browser capability is a deterministic evaluation primitive, not a general-purpose browser automation API.
